7. nettop 专项 — 唯一瓶颈，也是唯一壁垒

nettop 是 NetMonitor 独有的 CPU 开销来源。在 v3 测试中： 4/30 轮活跃，均值 11.7%，单次最高 125.3%。主进程其余 26/30 轮 CPU 接近零。

状态轮数CPU 范围说明
nettop 空闲26/30 (86.7%)0.0 - 9.2% (主进程)仅 UI 刷新 + SQLite flush
nettop 活跃4/30 (13.3%)55.3 - 125.3%每次持续 ~1 秒

根源：nettop 底层调的是 DTrace，每次运行需要遍历系统所有网络连接、解析 proc 表、做 DNS 反向查找。 这是 macOS 上获取"哪个 App 用了多少流量"唯一的公开 API。 替代方案（NEFilterDataProvider）已另有报告评估。

📋 一句话结论

NetMonitor 的优势
  • 主进程典型负载 0.2% — 四款最低
  • 唯一有进程流量 + 历史 + 图表
  • 包体最小 (6.9 MB)
  • 5s 间隔后 P95 降 49%，稳定性提升 30%
NetMonitor 的劣势
  • nettop 间歇爆发 (P95 125%) — 功能代价
  • 含 nettop 时均值 13.2%，统计上不如仅主进程干净
  • Stats 的 1.8σ 稳定性仍是标杆
📋 方法说明：四款软件从完全关闭状态同时启动，等 30s 热身完成后开始采样。ps -o pcpu= 每 2s 记录一次瞬时 CPU%，连续 60s 共 30 轮。内存/线程在采样结束后一次性采集。所有数据同等条件下获取，不进行任何加权或补偿。功能差异不参与性能评分。
⚠️ 局限性声明： · ps -o pcpu= 提供瞬时值，非连续监测。尖峰可能恰好落在采样窗口之间被遗漏或放大。
· 60s 采样足以捕捉稳态行为，但不足以评估长期趋势或内存泄漏。
· nettop 各次运行的 CPU 波动 ±30% 属于正常范围（取决于系统当前活跃连接数）。
· 所有 VSZ 38-39 GB 是 macOS mmap 的正常行为，不代表实际物理占用。
· 本报告不构成任何形式的推荐或排名建议，仅供数据分析参考。
