🔍 nettop 子进程深度分析

NetMonitor 是四款中唯一提供进程级流量监控的软件。这个功能通过 macOS 内置的 nettop 工具实现 — 每 3 秒启动一次 -L 1 单次快照（运行约 1 秒后自动退出）。

nettop CPU 特征

启动频率每 3 秒
单次运行时长~1 秒
单次 CPU46-123% (取决于系统负载)
等效持续 CPU~17% (duty cycle 33%)
空闲时 NetMonitor0.2-0.3% (极低)

为什么竞品不做

macOS 从系统层面不提供"每个进程用了多少网络"的公开 API。唯一替代方案是 NetworkExtension (NEFilterDataProvider)，需要用户手动批准系统扩展，开发复杂度是 nettop 方案的 10 倍以上。Stats / HagimiMonitor / NetWorker Pro 都选择跳过这个高成本功能。

⚠️ HagimiMonitor CPU 波动分析

HagimiMonitor 是四款中 CPU 表现最差的一个。其主进程 CPU 从 6.4% 到 30.1% 剧烈波动（标准差 8.1σ），是 NetMonitor 的 6.7 倍。

🏆 综合结论

NetMonitor 以显著优势胜出。它是四款中 CPU 最低、功能最全的唯一选择。

NetMonitor 优势

  • 主进程 CPU 最低 (1.7%) — 是 Stats 的 38%、NetWorker Pro 的 43%、Hagimi 的 13%
  • 唯一支持进程级流量监控
  • 唯一支持 SQLite 历史持久化（日/周/年）
  • 独家双击柱子查看进程详情
  • CPU 波动最小（仅次于 Stats）
  • nettop 开销可调（拉长间隔 → 进一步降低）

可改进方向

  • 内存 140 MB — 可加 SQLite LRU 缓存淘汰
  • nettop 间隔可从 3s → 5s 再降 40% CPU 开销
  • 长期运行后 accumulators 字典需要定期清理
