⚠️ 重要说明：nettop 是 macOS 唯一公开的进程级流量获取方式。 替代方案（NetworkExtension / NEFilterDataProvider）需要系统扩展 + 用户手动授权，开发复杂度 10 倍以上。 Stats、Hagimi、NetWorker Pro 均选择不实现此功能，因此不存在这笔开销。 是否保留此功能应基于用户需求而非性能数据。

7. HagimiMonitor 专项说明

HagimiMonitor 在本次测试中 CPU 表现最差：最低 5.3%、均值 9.7%、峰值 30.7%。它从不空闲（30 轮采样中没有任何一轮低于 5.3%）。

这可能与它较新的 UI 框架 / 渲染管线有关，也可能与本次测试的冷启动窗口有关。建议在正常运行 10+ 分钟后复测以排除初始化影响。

  • 主进程极低（中位数 0%、均值 1.7%）
  • 大部分时间完全空闲
  • 唯一有进程流量 + 历史数据
NetMonitor 劣势
  • nettop 间歇性爆发（P95 112%）
  • 含 nettop 时均值 10.5%
  • 内存比 Hagimi 多 21 MB
Stats 特点
  • 最稳定（σ=2.0、P95=6.2%）
  • 无功能额外开销
  • 但内存最高 (127 MB) · 线程最多 (16)
📋 方法局限性声明
· 采样精度：ps -o pcpu= 提供瞬时值，非连续监测。尖峰可能恰好落在采样窗口之间被遗漏。
· 采样时长：60 秒 (30 轮) 足以捕捉稳态行为，但不足以评估数小时运行趋势。
· 环境变量：未控制后台其他进程的 CPU 波动，可能存在 ±1-2% 的系统噪声。
· 公平性：所有软件同等条件冷启动 + 30s 热身。功能差异不参与性能评分。
· 建议：将此报告视为快照参考，非权威排名。
